\section{Work Evaluation}

Throughout the development process, the team has successfully implemented most of the core functionalities originally defined in the project scope. Compared to the initial project goals, the team was able to design and deliver the essential features of an online coding platform for Vietnamese students. However, the project still has some weaknesses that can be improved in the foreseeable future: 
\begin{itemize}
    \item Contest management has not been implemented due to its high complexity and limited development time. This module requires scheduling problems, handling time-limited submissions, calculating rankings, and managing access control, which could be addressed in future development phases.
    \item The chatbot can respond to basic user prompts, but its capability remains limited. Currently, it relies on simple keyword matching or scripted responses and uses a short-term memory mechanism, taking only the 20 most recent messages as context. As a result, it cannot fully understand the current conversation context. In future phases, the context memory mechanism can be improved to maintain longer conversation history and handle multi-turn interactions more effectively, providing a more coherent and helpful user experience.
\end{itemize}

These limitations are mainly due to the high complexity of the advanced features and the limited time and resources during development. Currently, the chatbot uses a short-term memory mechanism, considering only the 20 most recent messages as context, which prevents it from fully understanding the ongoing conversation. In future iterations, the team plans to enhance both the contest management module and the chatbot’s context memory, allowing the platform to handle longer conversation history, support multi-turn interactions more effectively, and provide a more comprehensive and engaging experience for Vietnamese students.
